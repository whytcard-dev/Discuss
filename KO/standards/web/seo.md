# SEO 표준 ## 핵심 원칙 - 사용자를 먼저 고려하고 검색 엔진은 그 다음에 고려 - 고품질의 가치 있는 콘텐츠 생성 - 기술적 우수성과 성능 보장 - 모바일 사용자를 우선으로 하여 구축 - 사용자 경험 측정 항목에 집중 - 가시성 향상을 위한 구조화된 데이터 구현 - 일관된 최적화 관행 유지 - 알고리즘 변경 사항 모니터링 및 적응 ## 기술적 SEO ### 페이지 구조 - 적절한 HTML5 의미 구조 구현 - 페이지당 하나의 H1 태그 사용 - 논리적 제목 계층 구조 생성(H1 → H2 → H3) - 설명적이고 키워드가 풍부한 제목 포함 - 의미적 HTML 요소 사용(`<article> `, `<section> `, `<nav> ` 등) - 콘텐츠 유형에 대한 schema.org 마크업 구현 - 중요한 콘텐츠는 JavaScript가 아닌 HTML로 유지 - JavaScript 없이도 콘텐츠에 액세스할 수 있도록 보장 ### 메타 태그 - 고유하고 설명적인 제목 태그(50-60자) 생성 - 설득력 있는 메타 설명 작성(120-155자) - 중복 콘텐츠를 방지하기 위한 표준 태그 구현 - 적절한 언어 속성(`) 사용<html lang="en"> `) - Open Graph 및 Twitter Card 메타 태그 구현 - 반응형 디자인을 위한 뷰포트 메타 태그 포함 - 메타 로봇 태그 적절히 사용 - 링크에 대한 rel 속성(nofollow, ugc, sponsored) 구현 ### URL - 짧고 설명적인 URL 만들기 - 하이픈을 사용하여 단어 구분 - 자연스러운 경우 관련 키워드 포함 - 논리적인 사이트 구조 유지 - 적절한 URL 매개변수 처리 구현 - 가능한 경우 URL 매개변수 사용 방지 - URL에 소문자 사용 - URL에 특수 문자 사용 방지 ### 성능 - Lighthouse 성능 측정 항목에서 90점 이상 달성 - Core Web Vitals 최적화: - LCP(Largest Contentful Paint): &lt; 2.5초 - FID(First Input Delay): &lt; 100ms - CLS(Cumulative Layout Shift): &lt; 0.1 - 적절한 이미지 최적화 구현 - CSS 및 JavaScript 최소화 - 브라우저 캐싱 활용 - 압축 활성화(Gzip, Brotli) - 서버 응답 시간 단축 - 중요한 CSS 구현 - 이미지 및 비디오에 대한 지연 로딩 사용 ### 인덱싱 및 크롤링 - 생성 및 XML 사이트맵 유지 관리 - robots.txt 파일 구현 - Google Search Console 및 Bing 웹마스터 도구 설정 - 크롤링 통계 및 오류 모니터링 - 깨진 링크 및 404 오류 수정 - 적절한 상태 코드 구현 - 언어/지역 변형에 rel=&quot;alternate&quot; 사용 - 대규모 사이트의 크롤링 예산 제어 ## 콘텐츠 최적화 ### 키워드 전략 - 철저한 키워드 조사 수행 - 키워드 밀도보다 검색 의도에 집중 - 특정 콘텐츠에 대한 롱테일 키워드 타겟팅 - 주제별 관련 키워드 그룹화 - 경쟁 키워드 분석 - 관련성 있는 경우 로컬 SEO 키워드 고려 - 키워드 순위 및 성과 추적 - 키워드 전략을 정기적으로 업데이트 ### 콘텐츠 생성 - 독창적이고 가치 있는 콘텐츠 생성 - 주제에 적합한 콘텐츠 길이 보장 - 자연스럽게 타겟 키워드 포함 - 관련 용어 및 동의어 사용 - 명확한 제목 및 부제목으로 콘텐츠 구성 - 시각적 콘텐츠(이미지, 비디오, 인포그래픽) 포함 - 정기적으로 콘텐츠 업데이트 - 얇거나 오래된 콘텐츠 제거 또는 개선 ### 온페이지 최적화 - 중요한 요소에 키워드 포함: - 제목 태그 - H1 제목 - URL - 첫 번째 단락 - 부제목(자연스러운 경우) - 설명적인 파일 이름과 대체 텍스트로 이미지 최적화 - 설명적인 앵커 텍스트로 내부 링크 구현 - 외부 링크에 설명적인 앵커 텍스트 생성 - 긴 콘텐츠에 목차 사용 - 질문 기반 콘텐츠에 FAQ 스키마 구현 - 추천 스니펫 최적화 - 행동 촉구 요소 포함 ## 모바일 최적화 - 반응형 디자인 구현 - 모바일 친화적 탐색 보장 - 탭 대상 최적화(최소 44×44px) - 렌더링 차단 리소스 제거 - 모바일 기기의 리소스 부하 최소화 - 여러 모바일 기기에서 테스트 - 확대/축소 없이 읽을 수 있는 글꼴 크기 보장 - 방해가 되는 삽입 광고 방지 ## 사용자 경험 - 매력적인 콘텐츠로 이탈률 최소화 - 페이지 로드 속도 개선 - 직관적 탐색 보장 - 브레드크럼 구현 - 명확한 행동 촉구 생성 - 접근성 보장(WCAG 준수) - 방해가 되는 광고 및 팝업 최소화 - 체류 시간 및 참여 최적화 ## 로컬 SEO - Google 비즈니스 프로필 생성 및 최적화 - NAP(이름, 주소, 전화번호) 일관성 보장 - 로컬 비즈니스 스키마 마크업 구현 - 위치별 페이지 생성 - 로컬 키워드 최적화 - 고객 리뷰 장려 - 관련 디렉토리에 비즈니스 나열 - 로컬 콘텐츠 생성 ## 전자 상거래 SEO - 제품 스키마 마크업 구현 - 고유한 제품 설명 생성 - 대체 텍스트가 있는 고품질 제품 이미지 사용 - 고객 리뷰 포함 - 카테고리 페이지 최적화 - 품절된 제품을 적절히 처리 - 브레드크럼 탐색 구현 - 최적화된 제품 URL 생성 ## 모니터링 및 분석 - 적절한 분석 추적 설정 - 유기적 검색 트래픽 모니터링 - 유기적 검색에서 전환율 추적 - 사용자 행동 메트릭 분석 - 키워드 순위 모니터링 - 정기적인 기술 SEO 감사 수행 - 백링크 프로필 추적 - 경쟁자 성과 모니터링 ## 고급 SEO ### 국제 SEO - 언어/지역 타겟팅을 위한 hreflang 태그 구현 - 적절한 URL 구조(ccTLD, 하위 도메인 또는 하위 디렉토리) 사용 - 로컬화된 콘텐츠 생성 - 더 나은 성능을 위해 로컬 호스팅 고려 - Search Console에서 지리적 타겟팅 구현 - 적절한 언어 메타 태그 사용 - SEO 전략에서 문화적 차이 고려 ### JavaScript SEO - 렌더링된 HTML에서 콘텐츠가 표시되는지 확인 - 필요한 경우 동적 렌더링 구현 - 적절한 지연 로딩 구현 사용 - &quot;렌더링된 소스 보기&quot;
