# معايير تحسين محركات البحث ## المبادئ الأساسية - التصميم للمستخدمين أولاً، ومحركات البحث ثانيًا - إنشاء محتوى عالي الجودة وقيم - ضمان التميز الفني والأداء - البناء لمستخدمي الهاتف المحمول كأولوية - التركيز على مقاييس تجربة المستخدم - تنفيذ بيانات منظمة لتحسين الرؤية - الحفاظ على ممارسات التحسين المتسقة - مراقبة تغييرات الخوارزمية والتكيف معها ## تحسين محركات البحث الفني ### بنية الصفحة - تنفيذ بنية دلالية HTML5 مناسبة - استخدام علامة H1 واحدة لكل صفحة - إنشاء تسلسل هرمي منطقي للعناوين (H1 → H2 → H3) - تضمين عناوين وصفية غنية بالكلمات الرئيسية - استخدام عناصر HTML الدلالية (`<article> `, `<section> `, `<nav> - تنفيذ ترميز schema.org لأنواع المحتوى - الاحتفاظ بالمحتوى المهم في HTML، وليس JavaScript - التأكد من إمكانية الوصول إلى المحتوى بدون JavaScript ### العلامات الوصفية - إنشاء علامات عنوان فريدة ووصفية (50-60 حرفًا) - كتابة أوصاف تعريفية مقنعة (120-155 حرفًا) - تنفيذ العلامات الأساسية لمنع المحتوى المكرر - استخدام سمات اللغة المناسبة (`<html lang="en"> - تنفيذ علامات التعريف الخاصة بـ Open Graph و Twitter Card - تضمين علامة التعريف الخاصة بـ Viewport للتصميم المستجيب - استخدام علامات التعريف الخاصة بـ Meta بشكل مناسب - تنفيذ سمات rel (nofollow و ugc و sponsored) للروابط ### عناوين URL - إنشاء عناوين URL قصيرة ووصفية - استخدام الواصلات لفصل الكلمات - تضمين الكلمات الرئيسية ذات الصلة عندما تكون طبيعية - الحفاظ على بنية موقع منطقية - تنفيذ معالجة معلمات URL المناسبة - تجنب معلمات URL عند الإمكان - استخدام الأحرف الصغيرة في عناوين URL - تجنب الأحرف الخاصة في عناوين URL ### الأداء - تحقيق أكثر من 90 درجة في مقاييس أداء Lighthouse - تحسين Core Web Vitals: - LCP (أكبر رسم محتوى): &lt; 2.5 ثانية - FID (تأخير الإدخال الأول): &lt; 100 مللي ثانية - CLS (تحول التخطيط التراكمي): &lt; 0.1 - تنفيذ تحسين الصورة المناسب - تصغير CSS و JavaScript - الاستفادة من تخزين المتصفح المؤقت - تمكين الضغط (Gzip و Brotli) - تقليل وقت استجابة الخادم - تنفيذ CSS الحرجة - استخدام التحميل الكسول للصور ومقاطع الفيديو ### الفهرسة والزحف - إنشاء خريطة موقع XML وصيانتها - تنفيذ ملف robots.txt - إعداد Google Search Console وأدوات مشرفي المواقع Bing - مراقبة إحصائيات الزحف والأخطاء - إصلاح الروابط المعطلة وأخطاء 404 - تنفيذ رموز الحالة المناسبة - استخدام rel=&quot;alternate&quot; للمتغيرات اللغوية / الإقليمية - التحكم في ميزانية الزحف للمواقع الكبيرة ## تحسين المحتوى ### استراتيجية الكلمات الرئيسية - إجراء بحث شامل عن الكلمات الرئيسية - التركيز على نية البحث بدلاً من كثافة الكلمات الرئيسية - استهداف الكلمات الرئيسية الطويلة لمحتوى معين - تجميع الكلمات الرئيسية ذات الصلة حسب الموضوع - تحليل الكلمات الرئيسية للمنافسين - مراعاة الكلمات الرئيسية لتحسين محركات البحث المحلية عند الاقتضاء - تتبع تصنيفات الكلمات الرئيسية والأداء - تحديث استراتيجية الكلمات الرئيسية بانتظام ### إنشاء المحتوى - إنشاء محتوى أصلي وقيم - ضمان طول المحتوى المناسب للموضوع - تضمين الكلمات الرئيسية المستهدفة بشكل طبيعي - استخدام المصطلحات والمرادفات ذات الصلة - هيكلة المحتوى بعناوين وعناوين فرعية واضحة - تضمين محتوى مرئي (صور ومقاطع فيديو ورسوم بيانية) - تحديث المحتوى بانتظام - إزالة أو تحسين المحتوى الرقيق أو القديم ### تحسين الصفحة - تضمين الكلمات الرئيسية في العناصر المهمة: - علامة العنوان - عنوان H1 - عنوان URL - الفقرة الأولى - العناوين الفرعية (حيث تكون طبيعية) - تحسين الصور باستخدام أسماء الملفات الوصفية ونص alt - تنفيذ الارتباط الداخلي باستخدام نص الرابط الوصفي - إنشاء نص الرابط الوصفي للروابط الخارجية - استخدام جدول المحتويات للمحتوى الطويل - تنفيذ مخطط الأسئلة الشائعة للمحتوى القائم على الأسئلة - تحسين المقتطفات المميزة - تضمين عناصر الحث على اتخاذ إجراء ## تحسين الهاتف المحمول - تنفيذ تصميم مستجيب - ضمان التنقل الصديق للهاتف المحمول - تحسين أهداف النقر (الحد الأدنى 44 × 44 بكسل) - القضاء على الموارد التي تحظر العرض - تقليل تحميل الموارد للأجهزة المحمولة - الاختبار على أجهزة محمولة متعددة - ضمان أحجام الخطوط القابلة للقراءة دون تكبير - تجنب الإعلانات البينية المتطفلة ## تجربة المستخدم - تقليل معدل الارتداد باستخدام محتوى جذاب - تحسين سرعة تحميل الصفحة - ضمان التنقل البديهي - تنفيذ فتات الخبز - إنشاء دعوات واضحة لاتخاذ إجراء - ضمان إمكانية الوصول (الامتثال لـ WCAG) - تقليل الإعلانات المنبثقة والإعلانات المتطفلة - تحسين وقت الانتظار والمشاركة ## تحسين محركات البحث المحلية - إنشاء ملف تعريف Google Business وتحسينه - ضمان اتساق NAP (الاسم والعنوان والهاتف) - تنفيذ ترميز مخطط الأعمال المحلي - إنشاء صفحات خاصة بالموقع - تحسين الكلمات الرئيسية المحلية - تشجيع مراجعات العملاء - إدراج الأعمال في الدلائل ذات الصلة - إنشاء محتوى محلي ## تحسين محركات البحث للتجارة الإلكترونية - تنفيذ ترميز مخطط المنتج - إنشاء أوصاف فريدة للمنتج - استخدام صور المنتج عالية الجودة مع نص بديل - تضمين مراجعات العملاء - تحسين صفحات الفئات - التعامل مع المنتجات غير المتوفرة في المخزون بشكل صحيح - تنفيذ التنقل الفتات الخبز - إنشاء عناوين URL للمنتجات المحسّنة ## المراقبة والتحليل - إعداد تتبع التحليلات المناسب - مراقبة حركة البحث العضوي - تتبع معدلات التحويل من البحث العضوي - تحليل مقاييس سلوك المستخدم - مراقبة تصنيفات الكلمات الرئيسية - إجراء عمليات تدقيق فنية منتظمة لتحسين محركات البحث - تتبع ملف تعريف الارتباط الخلفي - مراقبة أداء المنافسين ## تحسين محركات البحث المتقدم ### تحسين محركات البحث الدولية - تنفيذ علامات hreflang لاستهداف اللغة / المنطقة - استخدام بنية عنوان URL المناسبة (ccTLD أو النطاق الفرعي أو الدليل الفرعي) - إنشاء محتوى محلي - مراعاة الاستضافة المحلية لتحسين الأداء - تنفيذ الاستهداف الجغرافي في Search Console - استخدام علامات التعريف اللغوية المناسبة - مراعاة الاختلافات الثقافية في استراتيجية تحسين محركات البحث ### تحسين محركات البحث JavaScript - تأكد من ظهور المحتوى في HTML المُقدم - تنفيذ العرض الديناميكي إذا لزم الأمر - استخدام تنفيذ التحميل الكسول المناسب - الاختبار باستخدام أدوات &quot;عرض المصدر المُقدم&quot; - تقليل عرض جانب العميل للمحتوى المهم - تنفيذ رموز الحالة المناسبة لتنقل JavaScript - الاختبار باستخدام اختبار Google&#39;s Mobile-Friendly ### تحسين البحث الصوتي - تحسين الكلمات الرئيسية المحادثة - إنشاء محتوى الأسئلة الشائعة - التركيز على المقتطفات المميزة - تحسين سرعة الصفحة - تحسين الاستعلامات المحلية - استخدام البيانات المنظمة - الكتابة بنبرة محادثة - التركيز على الكلمات الرئيسية للسؤال (من، ماذا، متى، أين، لماذا، كيف)
