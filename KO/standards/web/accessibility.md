# 웹 접근성 표준 ## 핵심 원칙(POUR) - **인지 가능**: 정보는 사용자가 인지할 수 있는 방식으로 제공되어야 함 - **작동 가능**: 사용자 인터페이스 구성 요소는 모든 사용자가 작동할 수 있어야 함 - **이해 가능**: 정보와 작동을 이해할 수 있어야 함 - **견고성**: 콘텐츠는 현재 및 미래 기술과 함께 작동할 수 있을 만큼 견고해야 함 ## 준수 수준 - 키보드 트랩을 피하세요 - 복잡한 인터페이스에 대한 키보드 단축키를 구현하세요 - 모든 기능을 키보드로만 테스트하세요 - 사용자 정의 구성 요소가 키보드 상호 작용 패턴을 지원하는지 확인하세요 - 표준 키보드 상호 작용(스페이스바, Enter, Escape, 화살표 키)을 지원하세요 ## 이미지 및 미디어 ### 이미지 - 모든 이미지에 대한 대체 텍스트 제공(`alt` 속성) - 검증 오류 발생 후 양식 상태 유지 - 자동 완성 속성 지원 - 적절한 입력 유형 사용(이메일, 전화, 날짜 등) ### 대화형 요소 - 모든 컨트롤에 접근 가능한 이름이 있는지 확인 - 충분히 큰 터치 대상 제공(최소 44x44px) - 필요한 경우 ARIA 역할, 상태 및 속성 구현 - 사용자 지정 구성 요소가 WAI-ARIA 작성 관행을 따르는지 확인 - 포커스 순서가 시각적 순서와 일치하는지 확인 - 사용 `tabindex`를 적절히 사용합니다(양수 값은 피합니다) ## 텍스트 및 타이포그래피 - 텍스트에 상대 단위(rem, em)를 사용합니다. - 콘텐츠 손실 없이 텍스트 크기를 최대 200%까지 조정할 수 있도록 합니다. - 단락 텍스트의 줄 높이를 최소 1.5로 유지합니다. - 줄 길이를 50-75자 사이로 유지합니다. -
