# Allgemeiner Styleguide für WhytCard ## Einleitung Dieses Dokument definiert den allgemeinen Programmierstil und die Best Practices für alle im WhytCard-Projekt verwendeten Sprachen und Technologien. Diese Standards gewährleisten Konsistenz, Wartbarkeit und Qualität im gesamten Code. ## Inhaltsverzeichnis 1. [Grundlegende Prinzipien](#fundamental-principles) 2. [Versionskontrolle](#version-control) ## Grundprinzipien ### Klarheit vor Cleverness Priorisieren Sie immer die Klarheit des Codes gegenüber cleveren oder komplexen Lösungen: - Schreiben Sie leicht verständlichen Code - Vermeiden Sie obskure Sprachfeatures, sofern nicht unbedingt nötig - Priorisieren Sie Lesbarkeit und Wartbarkeit vor Kürze ### Konsistenz Behalten Sie die Konsistenz im gesamten Code bei: - Folgen Sie etablierten Mustern im Projekt - Mischen Sie nicht unterschiedliche Stile oder Ansätze - Passen Sie beim Ändern von vorhandenem Code dessen Stil an ### DRY (Don&#39;t Repeat Yourself) Vermeiden Sie Code-Duplikation: - Extrahieren Sie wiederholten Code in Funktionen oder Klassen - Erstellen Sie gemeinsam genutzte Dienstprogramme für allgemeine Vorgänge - Verwenden Sie Vererbung oder Komposition, um Verhalten gemeinsam zu nutzen ### KISS (Keep It Simple, Stupid) Ziehen Sie einfache Lösungen komplexen vor: - Beginnen Sie mit dem einfachsten Ansatz, der funktioniert - Fügen Sie Komplexität nur hinzu, wenn nötig - Teilen Sie komplexe Probleme in einfachere Teile auf ### YAGNI (You Aren&#39;t Gonna Need It) Fügen Sie keine Funktionalität hinzu, bis sie notwendig ist: - Vermeiden Sie spekulative Features - Implementieren Sie nur, was jetzt benötigt wird - Refactoring bei Bedarf, nicht präventiv ## Namenskonventionen ### Allgemeine Regeln - Namen sollten beschreibend sein und den Zweck vermitteln - Verwenden Sie vollständige Wörter statt Abkürzungen (außer gebräuchlichen) - Bleiben Sie mit vorhandenen Benennungsmustern konsistent ### Groß-/Kleinschreibung Verwenden Sie für jede Sprache die entsprechende Groß-/Kleinschreibung: - **camelCase**: JavaScript/TypeScript-Variablen und -Funktionen - **PascalCase**: Klassen in den meisten Sprachen, React-Komponenten - **snake_case**: Python-Variablen und -Funktionen - **kebab-case**: HTML-Elemente, CSS-Klassen, Dateinamen - **UPPER_SNAKE_CASE**: Konstanten in den meisten Sprachen ### Benennungsbeispiele ```javascript // JavaScript const userProfile = { firstName: &#39;John&#39;, lastName: &#39;Doe&#39;, EMAIL_REGEX: /^[^@]+@[^@]+\.[^@]+$/, }; function calculateTotalPrice(items) { // Implementierung } class UserManager { // Implementierung } ``` ```python # Python user_profile = { &quot;first_name&quot;: &quot;John&quot;, &quot;last_name&quot;: &quot;Doe&quot;, &quot;EMAIL_REGEX&quot;: r&quot;^[^@]+@[^@]+\.[^@]+$&quot;, } def calculate_total_price(items): # Implementierungsdurchlauf class UserManager: # Implementierungsdurchlauf ``` ## Formatierung ### Einrückung - Verwenden Sie im gesamten Projekt eine einheitliche Einrückung - Für JavaScript/TypeScript/HTML/CSS: 2 Leerzeichen - Für Python: 4 Leerzeichen (PEP 8) - Mischen Sie niemals Tabulatoren und Leerzeichen ### Zeilenlänge - Begrenzen Sie Zeilen für die meisten Sprachen auf maximal 100 Zeichen - Befolgen Sie für Python PEP 8 (79 Zeichen für Code, 72 für Kommentare) - Brechen Sie lange Zeilen auf eine für die Sprache natürliche Weise um ### Abstände - Verwenden Sie Leerzeichen um Operatoren - Verwenden Sie Leerzeichen nach Kommas und Semikolons – Verwenden Sie einheitliche Abstände in Funktionsaufrufen und Deklarationen. ### Klammern und Blöcke – Öffnende Klammern in derselben Zeile für die meisten Sprachen (außer für Python-Funktionen und -Klassen) – Einheitliche Verwendung von Klammern, auch für einzeilige Blöcke – Richten Sie schließende Klammern am Anfang der öffnenden Zeile aus. ### Beispiele ```javascript // JavaScript-Funktion calculateDiscount(price, discountPercent) { if (price &lt;= 0) { return 0; } const discount = price * (discountPercent / 100); return discount; } const items = [ { name: &#39;Item 1&#39;, price: 10 }, { name: &#39;Item 2&#39;, price: 20 }, { name: &#39;Item 3&#39;, price: 30 }, ]; ``` ```python # Python def calculate_discount(price, discount_percent): if price &lt;= 0: return 0 discount = price * (discount_percent / 100) return discount items = [ {&quot;name&quot;: &quot;Item 1&quot;, &quot;price&quot;: 10}, {&quot;name&quot;: &quot;Item 2&quot;, &quot;price&quot;: 20}, {&quot;name&quot;: &quot;Item 3&quot;, &quot;price&quot;: 30}, ] ``` ## Dokumentation ### Codekommentare - Verwenden Sie Kommentare, um das &quot;Warum&quot; und nicht das &quot;Was&quot; zu erklären - Kommentieren Sie komplexe Algorithmen oder nicht offensichtliche Lösungen - Halten Sie Kommentare bei Codeänderungen auf dem neuesten Stand - Vermeiden Sie redundante oder offensichtliche Kommentare ### Funktions- und Methodendokumentation Dokumentieren Sie alle Funktionen und Methoden mit: - Beschreibung des Zwecks - Parameter und deren Typen -
