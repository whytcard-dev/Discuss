Questi standard garantiscono coerenza, manutenibilità e qualità in tutta la base di codice. ## Indice 1. [Principi fondamentali](#fundamental-principles) 2. [Controllo della versione](#version-control) ## Principi fondamentali ### Chiarezza anziché intelligenza Dai sempre priorità alla chiarezza del codice rispetto a soluzioni intelligenti o complesse: - Scrivi codice che sia facile da capire - Evita funzionalità del linguaggio oscure a meno che non siano necessarie - Dai priorità alla leggibilità e alla manutenibilità rispetto alla brevità ### Coerenza Mantieni la coerenza in tutta la base di codice: - Segui i modelli stabiliti nel progetto - Non mescolare stili o approcci diversi - Quando modifichi il codice esistente, adatta il suo stile ### DRY (Non ripeterti) Evita la duplicazione del codice: - Estrai il codice ripetuto in funzioni o classi - Crea utilità condivise per operazioni comuni - Usa l&#39;ereditarietà o la composizione per condividere il comportamento ### KISS (Keep It Simple, Stupid) Preferisci le soluzioni semplici a quelle complesse: - Inizia con l&#39;approccio più semplice che funziona - Aggiungi complessità solo quando necessario - Suddividi i problemi complessi in parti più semplici ### YAGNI (You Aren&#39;t Gonna Need It) Non aggiungere funzionalità finché non è necessario: - Evita funzionalità speculative - Implementa solo ciò che è richiesto ora - Rifattorizza quando necessario, non preventivamente ## Convenzioni di denominazione ### Regole generali - I nomi devono essere descrittivi e comunicare lo scopo - Usa parole complete piuttosto che abbreviazioni (tranne quelle comuni) - Sii coerente con i modelli di denominazione esistenti ### Stili maiuscole/minuscole Usa lo stile maiuscole/minuscole appropriato per ogni linguaggio: - **camelCase**: variabili e funzioni JavaScript/TypeScript - **PascalCase**: classi nella maggior parte dei linguaggi, componenti React - **snake_case**: variabili e funzioni Python - **kebab-case**: elementi HTML, classi CSS, nomi di file - **UPPER_SNAKE_CASE**: Costanti nella maggior parte dei linguaggi ### Esempi di denominazione ```javascript // JavaScript const userProfile = { firstName: &#39;John&#39;, lastName: &#39;Doe&#39;, EMAIL_REGEX: /^[^@]+@[^@]+\.[^@]+$/, }; function calculateTotalPrice(items) { // Implementazione } class UserManager { // Implementazione } ``` ```python # Python user_profile = { &quot;first_name&quot;: &quot;John&quot;, &quot;last_name&quot;: &quot;Doe&quot;, &quot;EMAIL_REGEX&quot;: r&quot;^[^@]+@[^@]+\.[^@]+$&quot;, } def calculate_total_price(items): # Passaggio di implementazione class UserManager: # Passaggio di implementazione ``` ## Formattazione ### Indentazione - Utilizzare un&#39;indentazione coerente in tutto il progetto - Per JavaScript/TypeScript/HTML/CSS: 2 spazi - Per Python: 4 spazi (PEP 8) - Non mischiare mai tabulazioni e spazi ### Lunghezza della riga - Limitare le righe a un massimo di 100 caratteri per la maggior parte dei linguaggi - Per Python, seguire PEP 8 (79 caratteri per il codice, 72 per i commenti) - Dividere le righe lunghe in un modo naturale per il linguaggio ### Spaziatura - Utilizzare spazi attorno agli operatori - Utilizzare spazi dopo virgole e punti e virgola - Utilizzare una spaziatura coerente nelle chiamate di funzione e nelle dichiarazioni ### Parentesi e blocchi - Aprire le parentesi sulla stessa riga per la maggior parte dei linguaggi (ad eccezione delle funzioni e delle classi Python) - Utilizzo coerente delle parentesi anche per i blocchi su una sola riga - Allineare le parentesi di chiusura all&#39;inizio della riga di apertura ### Esempi ```javascript // Funzione JavaScript calculateDiscount(price, discountPercent) { if (price &lt;= 0) { return 0; } const discount = price * (discountPercent / 100); return discount; } const items = [ { name: &#39;Item 1&#39;, price: 10 }, { name: &#39;Item 2&#39;, price: 20 }, { name: &#39;Item 3&#39;, price: 30 }, ]; ``` ```python # Python def calculate_discount(price, discount_percent): if price &lt;= 0: return 0 discount = price * (discount_percent / 100) return discount items = [ {&quot;name&quot;: &quot;Item 1&quot;, &quot;price&quot;: 10}, {&quot;name&quot;: &quot;Item 2&quot;, &quot;price&quot;: 20}, {&quot;name&quot;: &quot;Item 3&quot;, &quot;price&quot;: 30}, ] ``` ## Documentazione ### Commenti al codice - Descrizione dello scopo - Parametri e relativi tipi - Valore di ritorno e tipo - Eccezioni che possono essere generate - Esempi di utilizzo per funzioni complesse ### Intestazioni dei file Includere all&#39;inizio di ogni file: - Breve descrizione dello scopo del file - Informazioni sull&#39;autore/i - Data di creazione - Informazioni sulla licenza ### Esempi ```javascript /** * Calcola il prezzo totale degli articoli dopo l&#39;applicazione degli sconti. * * @param {Array<Object> } items -
