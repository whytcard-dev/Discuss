# Стандарти веб-доступності ## Основні принципи (POUR) - **Сприйнятливість**: Інформація має бути представлена користувачами у спосіб, який вони можуть сприйняти - **Керування**: Компоненти інтерфейсу користувача мають бути керованими всіма користувачами - **Зрозумілість**: Інформація та операції мають бути зрозумілими - **Надійність**: Контент має бути достатньо надійним для роботи з сучасними та майбутніми технологіями ## Рівні відповідності - **Цільова відповідність**: WCAG 2.1 AA (мінімум) - **Прагнення до**: WCAG 2.1 AAA, де це можливо - **Юридичні вимоги**: ADA, Розділ 508, EAA (ЄС), AODA (Канада) за потреби - **Регулярне тестування**: Автоматизоване та ручне тестування доступності ## Семантична структура ### Семантика HTML - Використовуйте відповідні елементи HTML за призначенням - Реалізуйте належну структуру заголовків (h1-h6) у логічному порядку - Використовуйте елементи орієнтирів (`<main> `, `<nav> `, `<header> `, `<footer> `, `<aside> `) - Реалізуйте належні структури списків (`<ul> `, `<ol> `, `<dl> `) - Використовуйте ` <button>` для інтерактивних елементів керування, а не div або span - Використовуйте ` <a>` для навігації до нових сторінок або ресурсів - Використовуйте `</a> <table>` для табличних даних з відповідними заголовками - Уникайте перевизначення рідної семантики за допомогою ARIA, якщо це не необхідно ### Структура документа - Включіть один `</table> <h1>` на сторінку - Реалізуйте логічну ієрархію заголовків - Використовуйте описові назви сторінок - Надайте посилання для пропуску для користувачів клавіатури - Групуйте пов&#39;язані елементи форми за допомогою ` <fieldset>` та ` <legend>` - Використовуйте <label>елементи ` ` для елементів керування форми - Реалізуйте належну мову документа (` <html lang="en">`) - Використовуйте ` <section>`, ` <article>` та інші семантичні контейнери ## Доступність клавіатури - Переконайтеся, що всі інтерактивні елементи доступні з клавіатури - Реалізуйте логічний порядок табуляції (розумно використовуйте tabindex) - Надайте видимі індикатори фокусу (покращіть стилі за замовчуванням) - Переконайтеся, що немає перехоплень клавіатури - Реалізуйте комбінації клавіш для складних інтерфейсів - Тестуйте всю функціональність лише за допомогою клавіатури - Переконайтеся, що користувацькі компоненти підтримують шаблони взаємодії з клавіатурою - Підтримуйте стандартні взаємодії з клавіатурою (пробіл, Enter, Escape, клавіші зі стрілками) ## Зображення та медіа ### Зображення - Надайте альтернативний текст для всіх зображень (атрибут `alt`) - Використовуйте порожній alt (`alt=&quot;&quot;`) для декоративних зображень - Надайте описовий alt-текст для Інформативні зображення - Використовуйте відповідні текстові альтернативи для складних зображень (діаграм, графіків) - Переконайтеся, що текст не вбудований у зображення - Використовуйте ` <figure>` та ` <figcaption>` для зображень із підписами - Підтримуйте достатню контрастність для важливих зображень ### Медіа - Надайте підписи до відео - Надайте транскрипти для аудіоконтенту - За потреби впроваджуйте аудіоописи до відео - Переконайтеся, що медіаплеєри доступні з клавіатури - Надайте елементи керування для паузи, зупинки та регулювання гучності - Уникайте автоматичного відтворення медіа зі звуком - Переконайтеся, що контент не блимає більше 3 разів на секунду ## Колір та контрастність - Забезпечте достатнє співвідношення контрастності кольорів: - 4,5:1 для звичайного тексту (AA) - 3:1 для великого тексту (AA) - 7:1 для звичайного тексту (AAA) - 4,5:1 для великого тексту (AAA) - Не покладайтеся лише на колір для передачі інформації - Надайте додаткові індикатори (значки, візерунки, текст) - Тестуйте інтерфейси у відтінках сірого - Підтримуйте режими високої контрастності - Переконайтеся, що індикатори фокусування мають достатню контрастність - Тестуйте за допомогою симуляторів дальтонізму - Використовуйте доступні комбінації кольорів ## Форми та інтерактивні елементи ### Форми - Пов’яжіть підписи з елементами керування форми (використовуючи `for`) атрибут) - Групуйте пов&#39;язані елементи форми за допомогою `</figcaption></figure></article></section></html></label></legend> <fieldset>` та ` <legend>` - Надавайте чіткі інструкції та повідомлення про помилки - Позначайте обов&#39;язкові поля (не лише кольором) - Перевіряйте введені дані та надавайте доступні повідомлення про помилки - Зберігайте стан форми після помилок перевірки - Підтримуйте атрибути автозаповнення - Використовуйте відповідні типи введених даних (електронна пошта, телефон, дата тощо) ### Інтерактивні елементи - Переконайтеся, що всі елементи керування мають доступні назви - Надавайте достатньо великі цільові області дотику (мінімум 44×44 пікселів) - Реалізуйте ролі, стани та властивості ARIA за потреби - Переконайтеся, що користувацькі компоненти відповідають практикам створення WAI-ARIA - Надавайте зворотний зв&#39;язок для всіх взаємодій - Переконайтеся, що зміни стану оголошуються програмам зчитування з екрана - Тестуйте складні віджети за допомогою допоміжних технологій ## Керування фокусом - Підтримуйте видимі індикатори фокусу - Ніколи не видаляйте контури фокусу без альтернатив - Керуйте фокусом, коли вміст змінюється динамічно - Повертайте фокус на відповідні елементи після дій - Затримуйте фокус у модальних вікнах та діалогах - Реалізуйте програмний фокус для односторінкових програм - Переконайтеся, що порядок фокусування відповідає візуальному порядку - Використовуйте `tabindex` належним чином (уникайте позитивних значень) ## Текст та типографіка - Використовуйте відносні одиниці для тексту (rem, em) - Переконайтеся, що розмір тексту можна змінювати до 200% без втрата контенту - Підтримуйте висоту рядка щонайменше 1,5 для тексту абзацу - Довжина рядка має бути від 50 до 75 символів - Забезпечте достатній інтервал між абзацами - Використовуйте чітку, просту мову, коли це можливо - Уникайте вирівнювання тексту - Забезпечте належний контраст між текстом і фоном - Підтримуйте коригування інтервалів між текстом ## Реалізація ARIA - Використовуйте нативні елементи HTML перед ARIA, коли це можливо - Реалізуйте орієнтири ARIA для структури сторінки - Використовуйте ролі ARIA для визначення типів елементів, коли це необхідно - Застосовуйте стани та властивості ARIA для динамічного контенту - Оновлюйте стани ARIA під час зміни контенту - Тестуйте реалізації ARIA за допомогою програм зчитування з екрана - Дотримуйтесь практик авторингу WAI-ARIA для шаблонів - Уникайте конфліктуючих або надлишкових ARIA ## Динамічний контент та AJAX - Оголошуйте зміни динамічного контенту для програм зчитування з екрана - Належним чином використовуйте активні регіони ARIA - Керуйте фокусом під час оновлення контенту - Надавайте індикатори завантаження для асинхронного контенту - Забезпечте доступність користувацьких областей прокрутки - Зробіть односторінкові програми навігаційними - Зберігайте стан сторінки під час навігації назад/вперед - Забезпечте правильне оновлення історії браузера ## Тестування та валідація ### Автоматизоване тестування - Інтегруйте лінтинг доступності в робочий процес розробки - Використовуйте axe-core або аналогічні інструменти для автоматизованого тестування - Включіть тести доступності у конвеєрі CI/CD - Створення звітів про доступність - Відстеження проблем доступності з часом - Тестування за допомогою кількох автоматизованих інструментів - Розуміння обмежень автоматизованого тестування ### Ручне тестування - Проведення тестування лише з клавіатури - Тестування за допомогою програм зчитування з екрана (NVDA, JAWS, VoiceOver) - Тестування за допомогою збільшувальних екранів - Тестування за допомогою програмного забезпечення для розпізнавання мовлення - Перевірка контрастності кольорів вручну - Тестування зі збільшенням браузера до 200% - Тестування з різними розмірами дисплея ### Тестування користувачами - Залучення користувачів з інвалідністю до тестування - Тестування з користувачами допоміжних технологій - Збір відгуків щодо проблем доступності - Визначення пріоритетів виправлень на основі впливу - Документування способів вирішення відомих проблем - Проведення регулярних оглядів доступності -
