# Standarder för webbtillgänglighet ## Kärnprinciper (POUR) - **Uppfattbar**: Informationen måste vara presenterad för användare på ett sätt de kan uppfatta - **Användbar**: Användargränssnittskomponenter måste vara användbara för alla användare - **Förståeliga**: Information och funktion måste vara förståeliga - **Robust**: Innehållet måste vara tillräckligt robust för att fungera med nuvarande och framtida tekniker ## Efterlevnadsnivåer - **Mål för efterlevnad**: WCAG 2.1 AA (minimum) - **Sträva efter**: WCAG 2.1 AAA där det är möjligt - **Juridiska krav**: ADA, avsnitt 508, EAA (EU), AODA (Kanada) i tillämpliga fall - **Regelbunden testning**: Automatiserad och manuell tillgänglighetstestning ## Semantisk struktur ### HTML-semantik - Använd lämpliga HTML-element för deras avsedda syfte - Implementera korrekt rubrikstruktur (h1-h6) i logisk ordning - Använd landmärkeselement (``<main> `, `<nav> `, `<header> `, `<footer> `, `<aside> `) - Implementera korrekta liststrukturer (`<ul> `, `<ol> `, `<dl> `) - Använd ` <button>` för interaktiva kontroller, inte divs eller spans - Använd ` <a>` för navigering till nya sidor eller resurser - Använd `</a> <table>` för tabelldata med korrekta rubriker - Undvik att åsidosätta inbyggd semantik med ARIA om det inte är nödvändigt ### Dokumentstruktur - Inkludera en enda `</table> <h1>` per sida - Implementera logisk rubrikhierarki - Använd beskrivande sidtitlar - Tillhandahåll hopplänkar för tangentbordsanvändare - Gruppera relaterade formulärelement med ` <fieldset>` och ` <legend>` - Använd ` <label>` element för formulärkontroller - Implementera korrekt dokumentspråk ( <html lang="en">` `) - Använd ` <section>`, ` <article>` och andra semantiska behållare ## Tangentbordstillgänglighet - Se till att alla interaktiva element är tangentbordstillgängliga - Implementera logisk tabbordning (använd tabindex klokt) - Tillhandahåll synliga fokusindikatorer (förbättra standardstilar) - Se till att tangentbordsfällor undviks - Implementera tangentbordsgenvägar för komplexa gränssnitt - Testa all funktionalitet endast med tangentbord - Se till att anpassade komponenter stöder tangentbordsinteraktionsmönster - Stöd för standard tangentbordsinteraktioner (mellanslag, Enter, Escape, piltangenter) ## Bilder och media ### Bilder - Tillhandahåll alternativ text för alla bilder (`alt`-attribut) - Använd tom alt (`alt=&quot;&quot;`) för dekorativa bilder - Tillhandahåll beskrivande alt-text för informativa bilder - Använd lämpliga textalternativ för komplexa bilder (diagram, grafer) - Se till att text inte är inbäddad i bilder - Använd ` <figure>` och ` <figcaption>` för bilder med bildtexter - Bibehåll tillräcklig kontrast för viktiga bilder ### Media - Tillhandahåll bildtexter för videor - Tillhandahåll transkriptioner för ljudinnehåll - Implementera ljudbeskrivningar för videor vid behov - Se till att mediespelare är tangentbordsåtkomliga - Tillhandahåll kontroller för att pausa, stoppa och justera volym - Undvik att spela upp media automatiskt med ljud - Se till att inget innehåll blinkar mer än 3 gånger per sekund ## Färg och kontrast - Se till att färgkontrastförhållandena är tillräckliga: - 4,5:1 för normal text (AA) - 3:1 för stor text (AA) - 7:1 för normal text (AAA) - 4,5:1 för stor text (AAA) - Förlita dig inte enbart på färg för att förmedla information - Tillhandahåll ytterligare indikatorer (ikoner, mönster, text) - Testa gränssnitt i gråskala - Stödja högkontrastlägen - Se till att fokusindikatorer har tillräcklig kontrast - Testa med färgblindhetssimulatorer - Använd tillgängliga färgkombinationer ## Formulär och interaktiva element ### Formulär - Associera etiketter med formulärkontroller (med attributet `for`) - Gruppera relaterade formulärelement med `</figcaption></figure></article></section></html></label></legend> <fieldset>` och ` <legend>` - Ge tydliga instruktioner och felmeddelanden - Ange obligatoriska fält (inte bara med färg) - Validera inmatning och ge tillgängliga felmeddelanden - Bibehåll formulärets tillstånd efter valideringsfel - Stödja attribut för autofullständig ifyllning - Använd lämpliga inmatningstyper (e-post, tel, datum, etc.) ### Interaktiva element - Se till att alla kontroller har tillgängliga namn - Ge tillräckligt stora pekskärmar (minst 44×44px) - Implementera ARIA-roller, tillstånd och egenskaper vid behov - Se till att anpassade komponenter följer WAI-ARIA Authoring Practices - Ge feedback för alla interaktioner - Se till att tillståndsändringar meddelas till skärmläsare - Testa komplexa widgetar med hjälpmedel ## Fokushantering - Bibehåll synliga fokusindikatorer - Ta aldrig bort fokuskonturer utan alternativ - Hantera fokus när innehåll ändras dynamiskt - Återför fokus till lämpliga element efter åtgärder - Fånga fokus i modaler och dialogrutor - Implementera programmatiskt fokus för applikationer med en sida - Se till att fokusordningen matchar den visuella ordningen - Använd `tabindex` på lämpligt sätt (undvik positiva värden) ## Text och typografi - Använd relativa enheter för text (rem, em) - Se till att texten kan vara storleksändras upp till 200 % utan innehållsförlust - Bibehåll en radhöjd på minst 1,5 för stycketext - Håll radlängden mellan 50–75 tecken - Ge tillräckligt avstånd mellan stycken - Använd tydligt och enkelt språk när det är möjligt - Undvik justerad text - Säkerställ korrekt kontrast mellan text och bakgrund - Stöd för justeringar av textavstånd ## ARIA-implementering - Använd inbyggda HTML-element före ARIA när det är möjligt - Implementera ARIA-landmärken för sidstruktur - Använd ARIA-roller för att definiera elementtyper vid behov - Tillämpa ARIA-tillstånd och egenskaper för dynamiskt innehåll - Uppdatera ARIA-tillstånd när innehåll ändras - Testa ARIA-implementeringar med skärmläsare - Följ WAI-ARIA Authoring Practices för mönster - Undvik motstridiga eller redundanta ARIA ## Dynamiskt innehåll och AJAX - Meddela dynamiska innehållsändringar till skärmläsare - Använd ARIA-liveregioner på lämpligt sätt - Hantera fokus när innehåll uppdateras - Tillhandahåll laddningsindikatorer för asynkront innehåll - Säkerställ att anpassade rullningsområden är tillgängliga - Göra applikationer med en sida navigerbara - Bibehåll sidtillstånd under navigering bakåt/framåt - Säkerställ att webbläsarhistoriken är korrekt uppdaterad ## Testning och validering ### Automatiserad testning - Integrera tillgänglighetslinning i utvecklingsarbetsflödet - Använd axe-core eller liknande verktyg för automatiserad testning - Inkludera tillgänglighetstester i CI/CD-pipelinen - Generera tillgänglighetsrapporter - Spåra tillgänglighetsproblem över tid - Testa med flera automatiserade verktyg - Förstå begränsningarna med automatiserad testning ### Manuell testning - Utför tester endast med tangentbord - Testa med skärmläsare (NVDA, JAWS, VoiceOver) - Testa med skärmförstorare - Testa med programvara för taligenkänning - Verifiera färgkontrast manuellt - Testa med webbläsarzoom upp till 200 % - Testa med olika skärmstorlekar ### Användartester - Inkludera användare med funktionsnedsättningar i testning - Testa med användare av hjälpmedelsteknik - Samla in feedback om tillgänglighetsproblem - Prioritera korrigeringar baserat på påverkan - Dokumentera lösningar för kända problem - Genomför regelbundna tillgänglighetsgranskningar - Implementera kontinuerlig förbättring ## Dokumentation och utbildning - Dokumentera tillgänglighetsfunktioner - Tillhandahåll tillgänglighetsutlåtanden - Utbilda utvecklingsteamet i tillgänglighet - Inkludera tillgänglighet i designgranskningar - Dokumentera kända problem och färdplaner -
