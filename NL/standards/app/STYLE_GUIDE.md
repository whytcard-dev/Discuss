Deze standaarden garanderen consistentie, onderhoudbaarheid en kwaliteit in de gehele codebase. ## Inhoudsopgave 1. [Fundamentele principes](#fundamentele-principes) 2. [Versiebeheer](#versiebeheer) ## Fundamentele principes ### Duidelijkheid boven slimheid Geef altijd prioriteit aan duidelijke code boven slimme of complexe oplossingen: - Schrijf code die gemakkelijk te begrijpen is - Vermijd obscure taalfuncties, tenzij noodzakelijk - Geef prioriteit aan leesbaarheid en onderhoudbaarheid boven beknoptheid ### Consistentie Handhaaf consistentie in de hele codebase: - Volg vastgestelde patronen in het project - Meng geen verschillende stijlen of benaderingen - Pas bij het wijzigen van bestaande code de stijl aan ### DROOG (Don&#39;t Repeat Yourself) Vermijd duplicatie van code: - Extraheer herhaalde code in functies of klassen - Maak gedeelde hulpprogramma&#39;s voor algemene bewerkingen - Gebruik overerving of compositie om gedrag te delen ### KISS (Keep It Simple, Stupid) Geef de voorkeur aan eenvoudige oplossingen boven complexe: - Begin met de eenvoudigste aanpak die werkt - Voeg alleen complexiteit toe als dat nodig is - Verdeel complexe problemen in eenvoudigere delen ### YAGNI (You Aren&#39;t Gonna Need It) Voeg geen functionaliteit toe totdat het nodig is: - Vermijd speculatieve functies - Implementeer alleen wat nu nodig is - Refactor wanneer dat nodig is, niet preventief ## Naamgevingsconventies ### Algemene regels - Namen moeten beschrijvend zijn en het doel overbrengen - Gebruik volledige woorden in plaats van afkortingen (behalve algemene) - Wees consistent met bestaande naamgevingspatronen ### Hoofdlettergebruikstijlen Gebruik de juiste hoofdlettergebruikstijl voor elke taal: - **camelCase**: JavaScript/TypeScript-variabelen en -functies - **PascalCase**: Klassen in de meeste talen, React-componenten - **snake_case**: Python-variabelen en -functies - **kebab-case**: HTML-elementen, CSS-klassen, bestandsnamen - Gebruik consistente inspringing in het hele project - Voor JavaScript/TypeScript/HTML/CSS: 2 spaties - Voor Python: 4 spaties (PEP 8) - Meng nooit tabs en spaties ### Regellengte - Beperk regels tot maximaal 100 tekens voor de meeste talen - Volg voor Python PEP 8 (79 tekens voor code, 72 voor opmerkingen) - Breek lange regels op een manier af die natuurlijk is voor de taal ### Spatiëring - Gebruik spaties rond operatoren - Gebruik spaties na komma&#39;s en puntkomma&#39;s - Gebruik consistente spaties in functieaanroepen en declaraties ### Haakjes en blokken - Openingshaakjes op dezelfde regel voor de meeste talen (behalve Python-functies en -klassen) - Consistent gebruik van haakjes, zelfs voor blokken van één regel - Lijn sluitende haakjes uit met het begin van de openingsregel ### Voorbeelden ```javascript // JavaScript-functie calculateDiscount(prijs, kortingspercentage) { als (prijs &lt;= 0) { retourneer 0; } const korting = prijs * (kortingspercentage / 100); retourneer korting; } const items = [ { naam: &#39;Artikel 1&#39;, prijs: 10 }, { naam: &#39;Artikel 2&#39;, prijs: 20 }, { naam: &#39;Artikel 3&#39;, prijs: 30 }, ]; ``` ```python # Python def calculate_discount(price, discount_percent): if price &lt;= 0: return 0 discount = price * (discount_percent / 100) return discount items = [ {&quot;name&quot;: &quot;Artikel 1&quot;, &quot;price&quot;: 10}, {&quot;name&quot;: &quot;Artikel 2&quot;, &quot;price&quot;: 20}, {&quot;name&quot;: &quot;Artikel 3&quot;, &quot;price&quot;: 30}, ] ``` ## Documentatie ### Codeopmerkingen -
