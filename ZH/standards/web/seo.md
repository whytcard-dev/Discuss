# SEO 标准 ## 核心原则 - 首先为用户设计，其次为搜索引擎设计 - 创建高质量、有价值的内容 - 确保技术卓越性和性能 - 优先为移动用户构建 - 关注用户体验指标 - 实施结构化数据以增强可见性 - 保持一致的优化实践 - 监控并适应算法变化 ## 技术 SEO ### 页面结构 - 实施适当的 HTML5 语义结构 - 每页使用一个 H1 标签 - 创建逻辑标题层次结构（H1 → H2 → H3） - 包括描述性、关键字丰富的标题 - 使用语义 HTML 元素（`<article> `, `<section> `, `<nav> ` 等） - 为内容类型实施 schema.org 标记 - 将重要内容保留在 HTML 中，而不是 JavaScript - 确保无需 JavaScript 即可访问内容 ### 元标记 - 创建独特的描述性标题标记（50-60 个字符） - 编写引人注目的元描述（120-155 个字符） - 实施规范标记以防止重复内容 - 使用适当的语言属性（`<html lang="en"> `) - 实现 Open Graph 和 Twitter Card 元标记 - 包含视口元标记以进行响应式设计 - 适当使用元 robots 标记 - 为链接实现 rel 属性（nofollow、ugc、sponsored） ### URL - 创建简短、描述性的 URL - 使用连字符分隔单词 - 自然地包含相关关键字 - 维护合理的站点结构 - 实现适当的 URL 参数处理 - 尽可能避免使用 URL 参数 - 在 URL 中使用小写字母 - 避免在 URL 中使用特殊字符 ### 性能 - 在 Lighthouse 性能指标中获得 90 分以上 - 优化核心 Web 指标： - LCP（最大内容绘制）：&lt; 2.5 秒 - FID（首次输入延迟）：&lt; 100 毫秒 - CLS（累积布局偏移）：&lt; 0.1 - 实现适当的图像优化 - 最小化 CSS 和 JavaScript - 利用浏览器缓存 - 启用压缩（Gzip、Brotli） - 减少服务器响应时间 - 实现关键 CSS - 对图像和视频使用延迟加载 ### 索引和抓取 - 创建和维护 XML 站点地图 -实施 robots.txt 文件 - 设置 Google Search Console 和 Bing 网站站长工具 - 监控抓取统计信息和错误 - 修复断开的链接和 404 错误 - 实施正确的状态代码 - 对语言/区域变体使用 rel=&quot;alternate&quot; - 控制大型网站的抓取预算 ## 内容优化 ### 关键词策略 - 进行彻底的关键词研究 - 关注搜索意图而不是关键词密度 - 针对特定内容定位长尾关键词 - 按主题分组相关关键词 - 分析竞争对手的关键词 - 在相关时考虑本地 SEO 关键词 - 跟踪关键词排名和效果 - 定期更新关键词策略 ### 内容创作 - 创建原创、有价值的内容 - 确保内容长度适合主题 - 自然地包含目标关键词 - 使用相关术语和同义词 - 使用清晰的标题和副标题构建内容 - 包含视觉内容（图像、视频、信息图表） - 定期更新内容 - 删除或改进稀疏或过时的内容 ### 页面优化 - 在重要元素中包含关键词： - 标题标签 - H1 标题 - URL - 第一段 - 副标题（自然） - 使用描述性文件名和 alt文本 - 使用描述性锚文本实现内部链接 - 为外部链接创建描述性锚文本 - 对长内容使用目录 - 为基于问题的内容实现常见问题解答模式 - 针对精选摘要进行优化 - 包含号召性用语元素 ## 移动优化 - 实施响应式设计 - 确保移动友好型导航 - 优化点击目标（最小 44×44px） - 消除阻塞渲染的资源 - 最大限度地减少移动设备的资源负载 - 在多个移动设备上测试 - 确保无需缩放即可阅读的字体大小 - 避免侵入式插页式广告 ## 用户体验 - 通过引人入胜的内容最大限度地降低跳出率 - 提高页面加载速度 - 确保直观的导航 - 实施面包屑 - 创建清晰的号召性用语 - 确保可访问性（符合 WCAG 标准） - 最大限度地减少侵入性广告和弹出窗口 - 优化停留时间和参与度 ## 本地 SEO - 创建和优化 Google 商家资料 - 确保 NAP（名称、地址、电话）的一致性 - 实施本地商家模式标记 - 创建特定于位置的页面 - 针对本地关键字进行优化 - 鼓励客户评论 - 在相关目录中列出商家 - 创建本地内容 ## 电子商务 SEO - 实施产品架构标记 - 创建独特的产品描述 - 使用带有替代文本的高质量产品图片 - 包含客户评论 - 优化类别页面 - 妥善处理缺货产品 - 实施面包屑导航 - 创建优化的产品 URL ## 监控和分析 - 设置适当的分析跟踪 - 监控自然搜索流量 - 跟踪自然搜索的转化率 - 分析用户行为指标 - 监控关键字排名 - 定期执行技术 SEO 审核 - 跟踪反向链接配置文件 - 监控竞争对手的表现 ## 高级 SEO ### 国际 SEO - 实施 hreflang 标签以进行语言/地区定位 - 使用适当的 URL 结构（ccTLD、子域或子目录） - 创建本地化内容 - 考虑本地托管以获得更好的性能 - 在 Search Console 中实施地理定位 - 使用适当的语言元标记 - 在 SEO 策略中考虑文化差异 ### JavaScript SEO - 确保内容在呈现的 HTML 中可见 - 如果需要，实施动态渲染 - 使用适当的延迟加载实现 - 使用“查看渲染源”工具进行测试 - 最大限度地减少关键内容的客户端渲染 - 为 JavaScript 导航实施适当的状态代码 - 使用 Google 的移动友好测试 ### 语音搜索优化 - 针对对话关键词进行优化 - 创建常见问题解答内容 - 关注精选片段 - 提高页面速度 - 针对本地查询进行优化 - 使用结构化数据 - 以对话语气书写 - 关注问题关键词（谁、什么、何时、何地、为什么、如何）
