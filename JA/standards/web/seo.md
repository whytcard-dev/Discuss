# SEO 標準 ## コア原則 - ユーザーを第一に考え、検索エンジンを第二に考える - 高品質で価値のあるコンテンツを作成する - 技術的な卓越性とパフォーマンスを確保する - モバイルユーザーを優先して構築する - ユーザーエクスペリエンス指標に重点を置く - 可視性を高めるために構造化データを実装する - 一貫した最適化プラクティスを維持する - アルゴリズムの変更を監視して適応する ## 技術的な SEO ### ページ構造 - 適切な HTML5 セマンティック構造を実装する - ページごとに 1 つの H1 タグを使用する - 論理的な見出し階層を作成する (H1 → H2 → H3) - 説明的でキーワードが豊富な見出しを含める - セマンティック HTML 要素を使用する (`<article> `、`<section> `、`<nav> `など) - コンテンツタイプにschema.orgマークアップを実装する - 重要なコンテンツをJavaScriptではなくHTMLで保持する - コンテンツがJavaScriptなしでアクセス可能であることを確認する ### メタタグ - ユニークで説明的なタイトルタグを作成する(50〜60文字) - 魅力的なメタディスクリプションを書く(120〜155文字) - 重複コンテンツを防ぐために正規タグを実装する - 適切な言語属性を使用する(`<html lang="en"> `) - Open Graph と Twitter Card のメタタグを実装する - レスポンシブデザインのためにビューポートメタタグを含める - メタロボットタグを適切に使用する - リンクに rel 属性 (nofollow、ugc、sponsored) を実装する ### URL - 短く説明的な URL を作成する - 単語をハイフンで区切る - 自然な場合は関連キーワードを含める - 論理的なサイト構造を維持する - 適切な URL パラメータ処理を実装する - 可能な場合は URL パラメータを避ける - URL に小文字を使用する - URL に特殊文字を避ける ### パフォーマンス - Lighthouse のパフォーマンス指標で 90 以上のスコアを達成する - Core Web Vitals を最適化する: - LCP (Largest Contentful Paint): &lt; 2.5 秒 - FID (First Input Delay): &lt; 100 ミリ秒 - CLS (Cumulative Layout Shift): &lt; 0.1 - 適切な画像最適化を実装する - CSS と JavaScript を縮小する - ブラウザのキャッシュを活用する - 圧縮を有効にする (Gzip、Brotli) - サーバーの応答時間を短縮する - 重要な CSS を実装する - lazy画像と動画の読み込み ### インデックスとクロール - XMLサイトマップの作成と管理 - robots.txtファイルの実装 - Google Search ConsoleとBingウェブマスターツールの設定 - クロールの統計情報とエラーの監視 - 壊れたリンクと404エラーの修正 - 適切なステータスコードの実装 - 言語/地域のバリエーションにrel=&quot;alternate&quot;を使用する - 大規模サイトのクロールバジェットの管理 ## コンテンツの最適化 ### キーワード戦略 - 徹底的なキーワード調査の実施 - キーワード密度よりも検索意図に重点を置く - 特定のコンテンツのロングテールキーワードをターゲットにする - 関連キーワードをトピックごとにグループ化する - 競合他社のキーワードを分析する - 関連する場合はローカルSEOキーワードを検討する - キーワードのランキングとパフォーマンスを追跡する - キーワード戦略を定期的に更新する ### コンテンツ作成 - オリジナルで価値のあるコンテンツを作成する - トピックに適したコンテンツの長さを確保する - ターゲットキーワードを自然に含める - 関連用語と同義語を使用する - 明確な見出しと小見出しでコンテンツを構造化する - 視覚的なコンテンツ（画像、動画、インフォグラフィック）を含める - コンテンツを定期的に更新する - 内容の薄いコンテンツや古くなったコンテンツを削除または改善する ### オンページ最適化 - 重要なコンテンツにキーワードを含める要素: - タイトルタグ - H1 見出し - URL - 最初の段落 - 小見出し (自然な場合) - 説明的なファイル名と代替テキストで画像を最適化する - 説明的なアンカーテキストで内部リンクを実装する - 外部リンクに説明的なアンカーテキストを作成する - 長いコンテンツには目次を使用する - 質問ベースのコンテンツには FAQ スキーマを実装する - 強調スニペットを最適化する - 行動喚起要素を含める ## モバイル最適化 - レスポンシブデザインを実装する - モバイルフレンドリーなナビゲーションを確保する - タップターゲットを最適化する (最小 44×44 ピクセル) - レンダリングをブロックするリソースを排除する - モバイルデバイスのリソース負荷を最小限に抑える - 複数のモバイルデバイスでテストする - ズームせずに読みやすいフォントサイズを確保する - 邪魔なインタースティシャルを避ける ## ユーザーエクスペリエンス - 魅力的なコンテンツで直帰率を最小限に抑える - ページの読み込み速度を向上させる - 直感的なナビゲーションを確保する - パンくずリストを実装する - 明確な行動喚起を作成する - アクセシビリティを確保する (WCAG 準拠) - 邪魔な広告やポップアップを最小限に抑える - 滞在時間とエンゲージメントを最適化する ## ローカルSEO - Google ビジネス プロフィールを作成して最適化する - NAP（名前、住所、電話番号）の一貫性を確保する - ローカル ビジネス スキーマ マークアップを実装する - 地域固有のページを作成する - ローカル キーワードを最適化する - 顧客レビューを促進する - 関連ディレクトリにビジネスを掲載する - ローカル コンテンツを作成する ## e コマース SEO - 製品スキーマ マークアップを実装する - 独自の製品説明を作成する - 代替テキスト付きの高品質な製品画像を使用する - 顧客レビューを含める - カテゴリ ページを最適化する - 在庫切れの製品を適切に処理する - パンくずナビゲーションを実装する - 最適化された製品 URL を作成する ## 監視と分析 - 適切な分析トラッキングを設定する - オーガニック検索トラフィックを監視する - オーガニック検索からのコンバージョン率を追跡する - ユーザー行動の指標を分析する - キーワード ランキングを監視する - 定期的な技術的な SEO 監査を実行する - バックリンク プロファイルを追跡する - 競合他社のパフォーマンスを監視する ## 高度な SEO ### 国際 SEO - 言語/地域ターゲティング用の hreflang タグを実装する - 適切な URL 構造（ccTLD、サブドメイン、またはサブディレクトリ）を使用する - ローカライズされたコンテンツを作成する - パフォーマンスを向上させるためにローカル ホスティングを検討する - Search Console で地域ターゲティングを実装する - 適切な言語メタタグを使用する - SEO 戦略における文化の違いを考慮する ### JavaScript SEO - レンダリングされた HTML でコンテンツが表示されることを確認する - 必要に応じて動的レンダリングを実装する - 適切な遅延読み込み実装を使用する - 「レンダリングされたソースを表示」ツールでテストする - 重要なコンテンツのクライアント側レンダリングを最小限に抑える - JavaScript ナビゲーションに適切なステータス コードを実装する - Google のモバイル フレンドリー テストでテストする ### 音声検索の最適化 - 会話形式のキーワードに合わせて最適化する - FAQ コンテンツを作成する - 注目のスニペットに重点を置く - ページ速度を向上させる - ローカル クエリに合わせて最適化する - 構造化データを使用する - 会話調で書く - 質問キーワード（誰、何、いつ、どこで、なぜ、どのように）に重点を置く
